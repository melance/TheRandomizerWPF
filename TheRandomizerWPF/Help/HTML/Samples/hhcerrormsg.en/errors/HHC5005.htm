HHC5005: TOC Error: Level increments by more than one on line: "entry"

Problem: The compiler encountered a TOC entry that skips a level (indented by more then one level).

Result: The .chm file is corrupted and cannot be used.

Solution: This error only occurs when compiling a binary TOC—with a non-binary TOC, the entry is fixed up. If you don't specifically need a binary TOC, turn the option off in your project file. Otherwise, you must ensure that no level is indented more then one from the previous level.
