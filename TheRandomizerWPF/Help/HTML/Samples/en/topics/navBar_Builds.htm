Builds list

With help of this navigation panel, the Precision Helper can define and store lists of projects selected for batch compilation.

This application area is intended primarily for those users, who do not work in a more integrated company environment and who do not use one of the tools for task automation, compilation and distribution.
Nevertheless, even in such an environment may be this part of Precision Helper useful, for example, if you want to only compile a help projects of the complex product.

Working with Builds list

  • Predefined and stored builds are available in this navigation panel. By selecting the desired item (a build) you open the appropriate editor page with the list of projects used for this batch compilation.
  • Application offers two predefined builds - All projects and Recently opened. As their names suggest, these are the projects in your projects list, respectively the projects that was recently opened.
  • User defined (stored) builds are sorted in alphabetical order. A small info about the number of included projects is displayed right to the item.
  • You can rename or delete the stored build via the popup menu of the navigation panel. Of course, the "Remove" action does not delete the projects (neither the files, nor the project items from your projects list), but it is intended to remove user defined build only.
  • More informations about the build editor can be found in the Build (editor) help topic.

Note:
    If you get in a situation where you can see the navigation panel, but any appropriate page (list) is not visible, simply click the mouse on any item of navigation panel (or press the ENTER key if the panel is focused) and the page with a build item will be opened. If you want to open an editor for already selected build item, you have to use a double-click on it (or you have to press the ENTER key, if the navigation panel is focused).
